## 3. Data Flow

```
en.json / th.json ──(resources)──▶ i18next init (src/i18n/index.ts)
        ▲                              │  detector: localStorage → navigator → fallback 'en'
        │ parity test                  ├──▶ languageChanged ──▶ document.documentElement.lang
        │                              ▼
        └──────── useTranslation() in every component ──▶ rendered text

src/data/*.ts (ids, image paths, tech[], link, date) ──▶ section components ──▶ cards/badges
        └── translation KEY prefixes derived from ids (e.g. projects.items.${id}.title)

src/config/site.ts ──▶ Hero (resumeUrl), Contact (email/socials), Footer (name/socials)

ContactForm state ──▶ validateContact(values) ──▶ ContactFormErrors (keys) ──▶ t(key) at render
                 └──▶ simulated async submit ──▶ FormStatus idle|submitting|success|error
```

## 4. i18n Key Naming Convention

- Namespaced by section, camelCase leaves: `nav.about`, `hero.title`, `hero.subtitle`, `hero.downloadResume`, `about.heading`, `about.bio1`, `skills.heading`, `skills.categories.frontend`, `projects.heading`, `projects.viewDetails`, `projects.items.<id>.title`, `projects.items.<id>.description`, `certificates.items.<id>.name`, `gallery.items.<id>.caption`, `contact.form.labels.name`, `contact.form.errors.emailInvalid`, `contact.form.success`, `footer.copyright`, `a11y.openMenu`, `a11y.prevCertificate` …
- Item text keys are addressed by data-module id: adding an item = 1 data entry + keys in BOTH locale files (parity test enforces).
- Accessible-name strings live under `a11y.*` — they are user-visible to AT and MUST be translated like any text.
- `en.json` is the reference structure; `th.json` mirrors it exactly. No key may exist unreferenced (orphans removed when features change — constitution VIII).

## 5. Error / Edge-Case Strategy

- **No remote data exists**, so the explicit-UI-states rule (constitution VI) binds exactly two surfaces: (1) **ContactForm** — `FormStatus` union drives idle/submitting/success/error, all localized, error path deterministic for tests (simulated submit rejects when message contains a sentinel, e.g. exact string `"error"`, documented in code — otherwise resolves after ~900ms); (2) **images** — every `<img>` uses an `onError` handler that swaps to a styled neutral fallback (keeps layout, no broken icon), per FR-024.
- Unsupported browser language → `fallbackLng 'en'` (FR-004); stored preference always wins (detector order).
- Sticky-header anchor overlap → `scroll-mt-*` on every section (FR-009); smooth scroll only `motion-safe`.
- Long Thai text → `leading-relaxed` on body text, no letter-spacing/uppercase on Thai, `overflow-wrap: break-word` safety, verified at 320px in the polish pass (AC-9.2).
- Reduced motion → `MotionConfig reducedMotion="user"` (framer) + `motion-safe:` variants (CSS hover/scroll) — one mechanism per animation type, both centrally applied.
- jsdom gaps → `src/test/setup.ts` stubs IntersectionObserver/ResizeObserver/matchMedia before any framer-motion import runs in tests.

Testing rules: real i18n provider (no string mocks); `beforeEach` resets language to `en` + clears localStorage; queries prefer `getByRole` with accessible names; `user-event` over `fireEvent`; carousel `scrollBy` asserted via spy (jsdom doesn't implement scrolling).

## 7. Implementation Phases

1. **Scaffold** — package.json, index.html, vite/ts/eslint configs, minimal App; install; gates pass. (T001)
2. **Test infrastructure** — vitest config block, setup stubs, first trivial test green. (T002)
3. **i18n foundation (TDD)** — parity test red → full en/th dictionaries for ALL sections → init module + lang sync → green. (T003)
4. **Design system** — @theme tokens, fonts, base styles; UI primitives (SectionHeading, Badge, Reveal). (T004, T007)
5. **Content layer** — site config + typed data modules; placeholder SVGs + resume script. (T005, T006)
6. **Sections top-down** — Navbar+Switcher (TDD) → Hero → About/Skills → Projects → Certificates → Gallery → Contact (validateContact TDD) → Footer + App assembly + smoke. (T008–T015)
7. **Polish pass** — responsive 320px sweep (both languages), a11y audit against NFR-001, motion/reduced-motion verification, build + preview. (T016)

Per-task QC gate (every task, per constitution III): `npm run lint && npm run typecheck && npm test`.
